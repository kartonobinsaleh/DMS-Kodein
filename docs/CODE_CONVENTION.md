# Code Convention

## Naming

- camelCase → variables
- PascalCase → components
- kebab-case → file names

## Structure

- components/ → reusable UI
- app/ → pages
- lib/ → utilities
- store/ → Zustand

## API

- Keep logic in services
- Do not mix UI and business logic

## Validation

- Always use Zod

## Error Handling

- Use try-catch
- Return consistent error response
